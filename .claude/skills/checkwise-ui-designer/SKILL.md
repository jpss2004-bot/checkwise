# CheckWise UI Designer Skill

Use this skill for dashboards, forms, landing pages, report previews and frontend polish.

Before a full redesign, pair this skill with `checkwise-redesign-prep` and read:

- `docs/design-system/FRONTEND_REDESIGN_READINESS.md`
- `docs/design-system/VISUAL_REDESIGN_DOCTRINE.md`
- `docs/design-system/ASSET_MANIFEST.md`
- `docs/design-system/claude-design-v0.1/AUDIT.md`
- `frontend/app/globals.css`
- `frontend/tailwind.config.ts`

Treat Claude Design screenshots as references. Do not paste static HTML into the app.

## Design target

CheckWise should look like a premium legal-tech compliance platform.

It should feel:

- Executive.
- Calm.
- Precise.
- Trustworthy.
- Traceable.
- Professional.
- Not generic SaaS.
- Not student-project UI.

## Audit criteria

Evaluate:

- Visual hierarchy.
- Typography.
- Spacing.
- Contrast.
- Empty states.
- Component consistency.
- Dashboard usefulness.
- Form clarity.
- Accessibility.
- Responsiveness.
- Data visualization usefulness.
- Whether the UI explains risk and next action.

## Design rule

Every dashboard section should answer at least one of:

- What is missing?
- What is risky?
- Who owns it?
- What is due?
- What changed?
- What is the next action?

## Avoid

- Decorative charts.
- Fake metrics.
- Random gradients.
- Overcrowded cards.
- Low-contrast gray UI.
- Generic SaaS templates.
- Useless animations.
