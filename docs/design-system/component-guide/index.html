14 · Arquitectura

Shells de navegación

Un codebase con RBAC; cada rol entra a su superficie. Proveedor = sidebar; cliente/admin/plataforma = topbar. Admin y plataforma comparten ConsoleShell.

Portal Proveedor
Sidebar navy (surface-brand), BrandLogo xl, activo teal, icon-only colapsable, drawer móvil.
Consola Cliente
Topbar claro, chips (7+"Más"), campana, badge de plan, Wise. Banner break-glass ámbar si staff entra.
Admin / Operaciones
ConsoleShell compartido. Chip activo border-brand, nav scrollable + "Más", engrane fijo.
Plataforma
Mismo ConsoleShell, nav estrecha (usuarios, auditoría, feedback). Switcher a Operaciones.

Notas para móvil
  • El color es HSL nativo (triplet H S% L% envuelto en hsl()). Convierte a hex (tablas arriba) o consume HSL.
  • Mapea componentes nativos a la capa semántica (no a primitivas) para heredar cambios de paleta.
  • Touch target base = 40px (comfortable). Para móvil sube a ≥44px (Apple/Material).
  • Iconos: Phosphor (npm @phosphor-icons/react); el set de arriba es el realmente usado.
  • Estados de documento: reutiliza la tabla de §Estados literal — no inventes etiquetas.
Deriva: docs/DESIGN_SYSTEM.md (v1.0) tiene valores antiguos. Esta guía usa lo vigente de globals.css: expired = rojo (era naranja), in_review = azul (era navy), pending = gris (era ámbar), botón con 6 variantes.
